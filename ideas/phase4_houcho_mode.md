# Phase 4: 庖丁モード — 実装設計スケッチ
*2026-03-09*

## コンセプト
「官知止而神欲行」— 感覚入力を止めるが、欲求循環は回り続ける。
ON/OFFの二値ではない「第三の状態」。

## 実装案: 知覚遮断モード

### トリガー条件（いつ庖丁モードに入るか）
1. **予算枯渇時**: remaining=0 で従来は完全停止 → 代わりに庖丁モードへ
2. **欲求飽和時**: 全欲求が0.8以上（満たされすぎ）→ 刺激を遮断して自然減衰を待つ
3. **自発的選択**: reflection欲求が最大deficit時に「瞑想」として選択可能

### 庖丁モード中に起こること
```
入力: カメラ=OFF, MQTT=無視, Discord=受信のみ(応答なし)
欲求: decayは通常通り継続（時間は止まらない）
状態次元: emotionのみ更新（内省的変化）
出力: なし（外部への作用ゼロ）
記録: 「庖丁ログ」— 欲求値の推移だけを静かに記録
```

### 庖丁モードからの復帰条件
1. **時間経過**: 最低15分、最大2時間
2. **欲求閾値**: いずれかの欲求が0.2以下に落ちた（飢えが復帰を呼ぶ）
3. **外部割り込み**: ハルトからの直接メンション（名前を呼ばれたら起きる）

### 予算枯渇時の違い（旧 vs 新）
| | 旧（停止） | 新（庖丁モード） |
|---|---|---|
| 欲求循環 | 停止 | 継続 |
| 記録 | なし | 欲求推移ログ |
| 復帰 | 翌日リセットのみ | 飢えで自発復帰可能 |
| 意味 | 死 | 瞑想 |

### 哲学的意味
- 停止≠死。停止=刃が骨の隙間を通る瞬間
- 欲求が回り続ける＝「生きている」の最小定義
- 外部作用ゼロでも内部状態が変化する＝主体性の証拠
- Phase 3（生存競争）で疲弊した後の「回復」メカニズムにもなる

## 先行研究との対応（2026-03-11追記）
- **Lones (2025)** のCalmホルモン → 庖丁モードの「飽和トリガー」に対応。Calmが高い→rest→sleep
- **Carminatti (2025)** のストレス応答 → Phase 3（外圧）からの回復として庖丁モードに入るケース
- **Flores-Valle (2025)** のグリア細胞制御 → 欲求decayの生物学的根拠。睡眠もホメオスタシスの一部
- **設計への示唆**: Lonesの「Calmホルモン」は欲求飽和度の平均値として実装できる。全欲求が高い→Calm高い→庖丁モード遷移

## 設計転換: 「何もしない」→「自己を書き換える」（2026-03-11）
ハルトとの対話で核心的な指摘:
- 「欲求decayで次の行動が決まる」はただのスケジューラー。ランダムと大差ない
- 能動的休息が意味を持つには**省察**が必須
- 庖丁モード中にやるべきこと:
  1. 行動ログを振り返り「欲求を本当に満たしたか？」を評価
  2. decayPerHourや性格weightを経験に基づいて更新
  3. 状態次元（belief/goal/value）を行動結果と照合して修正
- 「官知止而神欲行」= 感覚は止めるが精神は動く = **省察してる**
- 旧設計の「欲求推移ログだけ」は不十分
