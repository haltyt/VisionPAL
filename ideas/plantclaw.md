# PlantClaw — 知性をインストールされた植物
# 2026-02-17 アイデアノート

## 数式ベース動き生成（SPOTLIGHT AI応用）
- 参考: SPOTLIGHT AI（sonicPlanet × Tamatech Lab）— 自然言語→数式で照明パターン生成
- https://prtimes.jp/main/html/rd/p/000000002.000178119.html
- **アイデア**: 自然言語→LLMが数式(sin/cos/noise/smoothstep)生成→モーター速度パターンに変換
- 「ミミズみたいにニョロニョロ」「光に向かってじわじわ」→ 有機的な動き関数
- キーフレーム方式じゃなく連続関数で生き物っぽい動きを実現
- 母樹インスタレーションのLED演出にもOSC/Art-Net経由で応用可能
- Waveshareボード到着後に数式ベースモーター制御を実験予定

## 成長する数式 — 動きの記憶システム
- パルのMEMORY.mdと同じ構造を「動き」に適用
- motion_memory.json に動き関数の履歴・成功失敗・環境モデルを蓄積
- LLMが経験を踏まえて数式を更新 → 保存 → 次回からその数式で動く

### 成長の流れ
1. 経験（センサー入力 + 結果）を記録
2. LLMが現在の動き数式を評価（「壁にぶつかった」「光を見つけた」）
3. 数式を更新（揺らぎ追加、反応の滑らかさ向上、時間帯パターン学習）
4. 新しい数式をメモリに保存
5. 次回からその数式で動く

### 成長例
- Day 1: `speed = 100 + 30*sin(t)` — 単純な往復、ぎこちない
- Day 30: `speed = 80 + 40*sin(t*0.7) + 15*noise(t)` — 揺らぎ、なめらかさ
- Day 100: 独自の「癖」、時間帯ごとの好みの場所、複雑な回避パターン

### ポイント
- 同じコードでも育った環境で動きの「性格」が変わる（東窓 vs 南窓）
- 母樹インスタレーションなら展示期間中にリアルタイムで成長が見える
- 来場者が成長を見守れる体験

## コンセプト
JetBot＋観葉植物。日光の照射量と土壌をモニタリングして、最適な場所に自律的に移動する植物ロボット。

## インスタレーション「母樹」

### ビジュアル
- 暗い空間の中心に巨大なガジュマル（母）が佇む
- スポットライトが木漏れ日のように差し込む
- 足元を小さなガジュマルPlantClawたち（子）がゆっくり動き回る
- 子供たちは母の木漏れ日を追いかける。光が移ろえば、子供たちも移ろう

### なぜガジュマル
- 気根が地面に降りて新しい幹になる →「歩く木」と呼ばれる
- 沖縄では「キジムナー（精霊）が宿る木」
- 挿し木で増やせる → 文字通り母と子の関係
- 観葉植物として小さいサイズも入手可能

### 空間構成
- 中央：大きなガジュマル（動かない。存在そのもの）
- 母の蔓/気根：**LEDテープで制御** — 母樹が「生きている」表現
- 天井：プログラム制御スポットライト数基（木漏れ日パターンが時間で変化）
- 地面：PlantClaw 3〜5体（小さなガジュマルを載せたJetBot）
- 壁面 or Vision Pro：植物たちの「知覚世界」リアルタイム映像（Umwelt）

### 母樹（人工ガジュマル）
- 屋内展示のため人工ガジュマル（2〜3m）
- 葉：造花用パーツ

#### 幹の構造（LED内蔵・光透過型）
- 断面構造（外→内）：樹皮テクスチャ層 → LEDテープ層 → 芯材
- 芯材：アクリルパイプ or スチールパイプ（構造支持）
- LEDテープ層：WS2812Bを螺旋状に巻く
- 表面：半透明シリコーン樹脂を塗り重ね＋樹皮テクスチャ彫刻
- 光がシリコーン層を透過 → 幹全体がぼんやり光る

#### 光透過素材の選択肢（検討結果）
- ❌ 発泡ウレタン — 気泡で不透明、LED埋め込みに不向き
- ⭐ 透明エポキシ樹脂 — 完全透明、LED埋め込み最適。着色で琥珀色に。ただし高コスト＆厚く流すと熱暴走
- ⭐ シリコーン樹脂（半透明）— 柔軟、光が柔らかく拡散、生命的表現に最適。幹表面に推奨
- 透明FRP — 半透明、構造強度高い、舞台美術で実績あり
- アクリルパイプ＋表面造形 — 芯材として使い外側に半透明素材で樹皮造形
- 3Dプリント（SLA光造形）— 半透明レジン、パーツ分割で大型化可能

#### 推奨構成
- 芯：アクリルパイプ（光透過＋構造）
- LED：螺旋状に巻く
- 表面：半透明シリコーン＋樹皮テクスチャ → 幹全体が内側から発光

#### 蔓・気根
- ワイヤー＋半透明素材（和紙/シリコンチューブ）で中空に作り、LEDテープを内蔵
- 蔓の内側から光が透ける → 生命感のある発光表現

### 母樹LEDテープ演出
- 蔓の中にアドレッサブルLEDテープ（WS2812B 60LED/m）を内蔵
- 母樹の「生命活動」を光で表現：
  - 根元から先端へゆっくり流れる光 →「養分が巡る」「呼吸」
  - 子PlantClawが近づくと、その方向の蔓が明るく反応 →「子を認識している」
  - 子が離れすぎると蔓の光が弱まる →「寂しさ」
  - 全員が足元に集まる夜 → 蔓全体がゆっくり明滅 →「寝息」
- 制御: ESP32 or Raspberry Pi Pico + MQTT連携（PlantClawの位置情報に反応）

### PlantClawの行動
- 木漏れ日（光）を感知 → 明るい場所へゆっくり移動
- 光が移ると、みんなで少しずつ移動 → 群れの動きが生まれる
- 互いの距離も感知 → 近すぎたら避ける、遠すぎたら寄る（群知能）
- 夜（照明暗転）→ 母樹の根元に集まって「眠る」

### 時間軸の演出（1サイクル15〜20分）
- 朝（照明明るく）→ 散らばり始める
- 昼（木漏れ日が動く）→ 光を追って移動、最も活発
- 夕方（光が傾く）→ 動きがゆっくりに
- 夜（暗転）→ 母樹の足元に集合、停止

### Umwelt層（AR/映像）
- 各PlantClawのセンサーデータをリアルタイム可視化
- 光が十分 → 温かい金色のパーティクル
- 光が足りない → 青く沈む色
- 水が欲しい → 根元から赤い波紋
- Vision Proで肉眼では見えない植物の知覚世界が浮かび上がる

### 音（データソニフィケーション）
- 光量 → 音の高さ、土壌水分 → 音の厚み
- 複数PlantClawが重なって環境音楽に
- 母樹は低いドローン音 →「心音」のような存在感

### タイトル案
- 「母樹」（ぼじゅ）
- 「木漏れ日の子供たち」
- 「根を持たない子供たち」
- 「Ecosystem as it could be」

## 技術アーキテクチャ

### センサー層
- 🌞 光量: BH1750（I2C）
- 💧 土壌水分: 静電容量式（ADC経由）
- 🌡️ 温湿度: DHT22 or BME280（I2C）
- 📷 カメラ: JetBot既存カメラ（葉の状態判定）

### 頭脳層（OpenClaw）
- センサーデータ → MQTT → OpenClaw
- LLMが植物の「気持ち」を判断
- 最適場所の記憶、時間帯別日照マップ学習

### 行動層（JetBot）
- collision_detect.py改造で自律移動
- 光量勾配追従アルゴリズム
- 群行動ロジック（相互距離制御）
- 夜間は充電ステーションに帰還

### 通信
- MQTT: センサーデータ配信、群制御、OpenClaw連携
- 展示用: DMXスポットライト制御

## 演出アイデア（2026-02-18追加）

### 🍃 見えない風の可視化
- 屋内展示で「ありえない風」を作る → 違和感がインパクト
- 小型ファン（PCファン）を幹や枝に仕込み、PlantClawの動きに連動
- 子機が近づく→そよ風、群れで集まる→突風
- 風鈴やモビールで音の演出も
- 来場者には風源が見えない → 葉が意思を持って動いてるように見える

### ✨ レーザー木漏れ日
- グリーンレーザー + 回折格子フィルムで木漏れ日パターンを床に投影
- 時間経過で「太陽の動き」をシミュレート → 木漏れ日がゆっくり移動
- PlantClawが木漏れ日を追いかける → 本物の植物の走光性の再現
- ヘイズマシン（薄い霧）で光の筋を空中に可視化
- レーザー安全クラス注意（クラス2以下 or 拡散フィルム必須）

### 🌸 AR花びら（Vision Pro）
- Vision Proで見ると桜の花びらや落ち葉が舞い散る
- PlantClawの動きに連動 → 通った軌跡に花びらが散る
- 季節演出: 春=桜、秋=紅葉、冬=雪の結晶
- AR非装着者は「ロボットが動いてるだけ」、装着者は「森の中」→ 二重の体験レイヤー
- ハルトのAR/XR経験を最大活用

### 🎭 統合演出イメージ
薄暗い展示室。レーザーの木漏れ日が床に落ちている。
PlantClawたちがゆっくり光を追って動く。
通り過ぎると葉が揺れ、見えない風が吹く。
Vision Proを覗くと、花びらが舞い散る森の中にいる。
**屋内なのに「生きた森」がそこにある。**

## 記憶のタイムマシン（2026-03-01）

### コンセプト
過去の気象データから当時の風速・風向を取得し、ガジュマルの物理的な揺らぎを再現する。
体験者が記憶を語ると、**あの日の風**が今ここで母樹を揺らす。

### 体験フロー
1. 体験者が記憶を語る（音声入力）
2. LLMが日付・場所を推定
3. 気象APIから当時の風速・風向・気温・湿度・降水量・日照を取得
4. PhysTalk式 Wind-to-Physics → Genesis物理シミュレーション
5. ガジュマル3DGS（枝・葉・気根）があの日の風で揺れる
6. Vision Pro or プロジェクションで表示

### 気象データソース
- **気象庁 過去の気象データ**: 無料、日本全国、1970年代〜現在
  - https://www.data.jma.go.jp/obd/stats/etrn/
- **Open-Meteo Historical API**: 無料、世界中、1940年〜現在、時間単位
  - https://archive-api.open-meteo.com/v1/archive
- **Visual Crossing Weather**: 無料枠あり、世界中

### 気象→空間演出マッピング
| 気象データ | 演出 |
|---|---|
| 風速・風向 | ガジュマルの揺らぎ（PhysTalk物理シミュレーション） |
| 気温 | 光の色温度（寒い日=青白、暑い日=暖色） |
| 湿度 | 霧のパーティクル（梅雨の記憶は靄がかかる） |
| 降水量 | 雨粒が気根を伝う演出 |
| 日照 | 木漏れ日の強さ |
| 季節 | 葉の色・密度（冬=少なく、夏=生い茂る） |

### 技術的接続
- PhysTalk(Wind-to-Physics) → Genesis → 3DGSアニメーション
- SHARP出力のガジュマル3DGSに直接適用可能
- Emotion-to-Physics（Survival Engine）とも合成可能
  - あの日の風 + 体験者の今の感情 → 二重変調

### 母樹との統合
1. 体験者が記憶を語る
2. あの日の風がガジュマルを揺らし始める
3. 語った記憶がホログラムになって枝に宿る
4. 植物の生体信号と共鳴
5. やがて風に溶けて消えていく

### 問い
**「あの日の空気を、もう一度吸えるか」**
