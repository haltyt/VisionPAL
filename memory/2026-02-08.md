## pal_speak.sh頭切れ修正

### 原因
- USBスピーカー(hw:4,0)がモノ→ステレオ変換(Route conversion)を挟んでいた
- start_threshold=24000(0.5秒)でバッファが溜まるまで再生開始しない
- 無音プリロールはHWがスキップ → 実信号が必要

### 解決
- 超低周波(20Hz)極小音量(0.002)プリロール0.3秒 + ステレオ変換
- `pasuspender -- aplay -D hw:4,0` でPulseAudio一時停止して直接再生
- USB autosuspend無効化（`echo -1 > /sys/module/usbcore/parameters/autosuspend`）

## Bluetoothスピーカー接続成功！🎉

### 環境
- 新BTアダプタ: CSR BT4.0 (0a12:0001) — USBスピーカーと差し替え
- Sony LSPX-S1 MAC: AC:9B:0A:AA:B8:F6

### 解決した問題（4つ）
1. **bluetoothctl「No default controller」** → `sudo bluetoothctl`で解決
2. **LEスキャンでLSPX-S1見えない** → `menu scan` → `transport bredr`で解決
3. **A2DP「Protocol not available」** → NVIDIAデフォルトの`--noplugin=audio,a2dp,avrcp`を削除（`/lib/systemd/system/bluetooth.service.d/nv-bluetooth-service.conf`）
4. **D-Bus AccessDenied** → `/etc/dbus-1/system.d/bluetooth.conf`にhaltytユーザーのallow追加、haltytをbluetoothグループに追加、PulseAudioの起動順序（bluetooth→PA）が重要

### 接続確認
- paplay経由でLSPX-S1から音声再生成功！
- PulseAudio sink: `bluez_sink.AC_9B_0A_AA_B8_F6.a2dp_sink`
