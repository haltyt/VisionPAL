## VisionPAL起動手順（正しい方法）

### JetBot (192.168.3.8)
```bash
# 1. MJPEGサーバー
ssh jetbot@192.168.3.8 "cd ~ && nohup python3 mjpeg_server.py --usb > /tmp/mjpeg.log 2>&1 &"
# 2. body_sensor（ファイルがなければscp）
ssh jetbot@192.168.3.8 "PYTHONUNBUFFERED=1 nohup python3 ~/body_sensor.py > /tmp/body_sensor.log 2>&1 &"
# 3. collision_detect
ssh jetbot@192.168.3.8 "PYTHONUNBUFFERED=1 nohup python3 ~/collision_detect.py > /tmp/collision.log 2>&1 &"
```

### Jetson (192.168.3.5)
```bash
# 最新コードをデプロイ
scp /home/node/.openclaw/workspace/VisionPAL/Cognition/*.py haltyt@192.168.3.5:/tmp/cognition/

# 4. vlm_watcher
ssh haltyt@192.168.3.5 "cd /tmp/cognition && PYTHONUNBUFFERED=1 nohup python3 vlm_watcher.py > /tmp/vlm.log 2>&1 &"

# 5. cognitive_loop（★GATEWAY_TOKEN必須！）
TOKEN=$(ssh haltyt@192.168.3.5 "python3 -c \"import json; d=json.load(open('/home/haltyt/.openclaw/openclaw.json')); print(d['gateway']['auth']['token'])\"")
ssh haltyt@192.168.3.5 "cd /tmp/cognition && PYTHONUNBUFFERED=1 OPENCLAW_GATEWAY_TOKEN=$TOKEN OPENCLAW_API_URL=http://172.19.0.2:18789 nohup python3 cognitive_loop.py --interval 5 --monologue-cooldown 10 > /tmp/cogloop.log 2>&1 &"
```

### 停止
```bash
ssh haltyt@192.168.3.5 "ps aux | grep -E '(cognitive_loop|vlm_watcher)' | grep -v grep | awk '{print \$2}' | xargs kill 2>/dev/null"
ssh jetbot@192.168.3.8 "ps aux | grep -E '(body_sensor|collision_detect|mjpeg_server)' | grep -v grep | awk '{print \$2}' | xargs kill 2>/dev/null"
```

### ハマりポイント
- **OPENCLAW_GATEWAY_TOKEN** と **OPENCLAW_API_URL** を環境変数で渡さないと401エラー
- トークンはJetsonの `~/.openclaw/openclaw.json` → `gateway.auth.token` から取得
- APIのURLは `http://172.19.0.2:18789`（コンテナIP）
- TTSファイルはコンテナ内に生成される → `docker cp` でホストに取り出す（cognitive_loop.py修正済み）
- body_sensor.pyはJetBotの~/に無いことがある → scpでデプロイ
- JetBotのnohup起動は `PYTHONUNBUFFERED=1` 必須
