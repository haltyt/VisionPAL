# 🔬 夜間リサーチ 2026-02-11
## テーマ: マルチモーダルAI（視覚+音声+対話）

---

### 1. 📄 Situated Embodied Human-Robot Conversation with Real-Time Multimodal LLMs
- **出典**: arXiv 2026年2月3日（MIT Media Lab / CMU）
- **概要**: リアルタイムマルチモーダルLLMとツール呼び出しを組み合わせた、ロボットとの対面対話システム。視覚・音声・対話を統合した「situated conversation」のレシピ。
- **パル的注目点**: まさにパルがやりたいこと！カメラで見て、聞いて、喋る。ツール呼び出し統合はOpenClawと相性良さそう。
- **リンク**: https://arxiv.org/search/?query=small+multimodal+model+robot+embodied (トップ結果)

### 2. 📄 NORA: Small Open-Sourced Vision Language Action Model
- **出典**: arXiv 2025年4月
- **概要**: 小型のオープンソースVLA（Vision-Language-Action）モデル。ゼロショットでロボットタスク実行＋推論可能。視覚エンコーディングの改善でオブジェクト把持精度向上。
- **パル的注目点**: 「小型」がポイント。Jetsonで動く可能性あり。JetBotの自律行動に使えるかも。

### 3. 📄 ObjEmbed: Universal Multimodal Object Embeddings
- **出典**: arXiv 2026年2月2日
- **概要**: 画像内の個別オブジェクトごとに領域埋め込みを生成。1回のフォワードパスで全オブジェクト+画像全体をエンコード。18ベンチマークで高性能。
- **パル的注目点**: パルの「目」で見たものを理解する精度向上に直結。物体認識→対話のパイプラインに使える。

### 4. 🔥 UI-TARS-2 (ByteDance) - Multimodal Agent
- **出典**: GitHub bytedance/UI-TARS（2025年9月 v2リリース）
- **概要**: GUIタスク、ゲーム、コード、ツール使用を統合した「All In One」エージェントモデル。Qwen 2.5VLベース、7Bモデルがオープンソース。強化学習で推論能力強化。
- **パル的注目点**: 7Bモデルなのでエッジ寄り。マルチモーダルエージェントのアーキテクチャ参考に。

### 5. 📄 Edge General Intelligence: Knowledge Distillation for Mobile Agentic AI
- **出典**: arXiv 2025年11月
- **概要**: 大規模モデルの知識をエッジデバイス向けに蒸留するパラダイム「EGI」。モバイル/エッジでのエージェントAI実現を目指す。
- **パル的注目点**: Jetson NanoみたいなエッジデバイスでマルチモーダルAIを動かすための蒸留手法。まさに必要な技術。

## 💡 パル＆ハルトのJetson環境で試せそうなアイデア

### アイデア1: 「見て喋るパル」マルチモーダル対話強化
現在のvoice_monitorは音声→テキスト→LLM→TTS。ここにカメラ映像の理解を統合する。
- **やり方**: 音声入力時にカメラスナップも取得 → OpenClaw APIに画像+テキストを同時送信 → 「今見えてるもの」について会話
- **必要なもの**: voice_monitor.pyに画像キャプチャ追加、OpenClaw APIのマルチモーダル入力対応
- **難易度**: ★★☆（既存パーツの組み合わせでいける）

### アイデア2: NORA風VLAでJetBot自律タスク
NORAのような小型VLAモデルをJetBotに載せて「あのボールを取ってきて」的な指示実行。
- **やり方**: 小型VLAモデル（3B以下）をJetsonで量子化実行 → カメラ入力→行動出力パイプライン
- **課題**: Jetson Nanoのメモリ4GBが厳しい。Jetson Orinなら余裕だが…
- **難易度**: ★★★（モデルサイズがボトルネック、要検証）
