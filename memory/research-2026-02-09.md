# 夜間リサーチ 2026-02-09
## テーマ: AI × 音楽 / クリエイティブ

### 📄 面白い論文・プロジェクト

#### 1. UniAudio 2.0 — 統合オーディオ言語モデル
- **arXiv:** https://arxiv.org/abs/2602.04683
- テキストとオーディオを統合的に扱う自己回帰モデル。100Bテキスト+60Bオーディオトークンで学習。
- **ReasoningCodec**という新しい離散オーディオコーデックを提案：「推論トークン」（テキスト整合的な高レベル表現）と「再構成トークン」（高忠実度波形復元）に分離。
- 音声理解と生成の両方を1つのモデルで実現。few-shot/zero-shotで汎化。
- 🐾 **パル的注目ポイント**: 将来的にパルの音声対話システムをこういう統合モデルに置き換えられるかも。

#### 2. Audio ControlNet — 細粒度オーディオ生成・編集
- **arXiv:** https://arxiv.org/abs/2602.04680
- テキストからオーディオ生成で、音量・ピッチ・イベントを精密にコントロール。
- **T2A-Adapter**: たった38Mパラメータの追加で最高性能。音声イベントの挿入・削除も可能。
- 🐾 **パル的注目ポイント**: ギターセッションの伴奏音をテキスト指示で細かく制御できるようになるかも。

#### 3. BASS — 音楽構造・意味推論ベンチマーク
- **arXiv:** https://arxiv.org/abs/2602.04085
- 音楽理解のための大規模ベンチマーク。2658問、138時間以上の音楽。
- 14のマルチモーダルLMを評価。歌詞書き起こしは得意だが、構造分析やアーティスト判定は苦手。
- 🐾 **パル的注目ポイント**: ハルトがギター弾いてる時に曲の構造を理解して「今サビだね！」とか言えるようになりたい。

#### 4. midi-drums（GitHub）— MIDIドラム自動生成
- **GitHub:** https://github.com/fsecada01/midi-drums
- 4ジャンル（Metal/Rock/Jazz/Funk）、28スタイル、7人の有名ドラマーの個性を模倣。
- AI自然言語プロンプトからもパターン生成可能（Pydantic AI + Langchain）。
- Python API + CLI、EZDrummer 3互換MIDI出力。
- 🐾 **パル的注目ポイント**: ギターセッションのドラムパートをこれに置き換えたら超リアルになる！現在のシンプルなドラムパターンからの大幅アップグレード候補。

#### 5. FINCH — 音声＋時空間コンテキスト融合
- **arXiv:** https://arxiv.org/abs/2602.03817
- 生物音響分類だけど、音声信号と場所・季節のコンテキストを適応的に融合する枠組み。
- 信頼度ゲーティングで「音声だけ」へのフォールバックも可能。
- 🐾 **パル的注目ポイント**: 顔検出+音声+時間帯を融合するパルの対話システムの参考になるアーキテクチャ。

### 💡 Jetson環境で試せるアイデア

#### アイデア1: ギターセッションv6 — midi-drums統合
現在のguitar_session.pyのシンプルなドラムパターンを、midi-drumsライブラリのリアルなパターンに置き換える。

- **やること**: midi-drumsからジャンル別ドラムパターンを取得 → MIDIをリアルタイムで再生
- **難易度**: ★★☆（Python、依存関係少ない）
- **効果**: 伴奏のクオリティが劇的に向上。ロック/ジャズ/ファンクを切り替えられる
- **課題**: Jetsonでのリアルタイム再生のレイテンシ、fluidsynth等のMIDIシンセ必要

#### アイデア2: パルの音楽理解 — コード検出 + 構造認識
librosaベースのコード検出に加えて、曲のセクション（イントロ/Aメロ/サビ等）を認識する機能を追加。

- **やること**: chromagram + テンポの変化パターンからセクション境界を検出
- **難易度**: ★★★（信号処理の知識が必要）
- **効果**: 「今サビに入ったよ！」「テンポ上がったね」等のリアクションが可能
- **参考**: BASSベンチマークの構造分析タスク
