### DualSense操縦スクリプト (dualsense_drive.py)
- Jetsonホスト: ~/dualsense_drive.py → MQTT → JetBot mqtt_robot.py
- 左スティック操縦、斜め移動（差動操舵）、R2ブースト、×緊急停止、○終了
- **差動操舵対応**: mqtt_robot.pyにleft_speed/right_speed対応パッチ適用済み
- ハプティクス問題: **hidrawをopen（読み書き/書き込み専用問わず）するとjs0の入力が止まる**
  - 振動テスト単体は成功（Layout A: flag1=0x03, flag2=0x15, offset 4,5, CRC32付き）
  - しかしjs0と同時にhidraw1を開くと入力が来なくなる
  - さらに不正レポート送信後、DualSense自体が入力送信を停止した可能性
  - **要: DualSense裏面リセットボタンで完全リセット → 再ペアリング**
  - 振動は別プロセスで実装するか、USB有線接続に切り替える必要あり
- ハプティクスなし版は動作確認済み（斜め移動含む）

### 午後セッション（ハルトと）
- DualSense BT接続後、`Enabled=0`問題を発見 — kernelがHIDデバイスを無効化する
- 毎回 `echo 1 | sudo tee /sys/.../input/inputXXXX/enabled` が必要
- hidraw1(振動)を有効にして再テスト → やはりjs0入力が完全停止。kernel 4.9では不可
- **USB有線接続で解決する方針** — USB-C to A変換が必要、明日実施予定
- TOOLS.mdにDualSense起動手順・Enabled問題・振動問題を追記済み

### 重要な教訓
- DualSense BT: `Enabled=0`問題 — BT再接続のたびにkernelがinputデバイスを無効にする。手動有効化必須
- DualSense BT: hidrawへの書き込みがjs0入力と干渉する（kernel 4.9, hid-playstationドライバーなし環境）
- DualSense BT: 不正なHIDレポート送信でコントローラーが応答しなくなる可能性 → 裏面リセットで復旧
- Jetson Nano kernel 4.9にはhid-playstationドライバーがない → モーションセンサー/振動の正式サポートなし
- JetBotのbluetoothサービスはcore-dumpで落ちやすい → `sudo /usr/lib/bluetooth/bluetoothd -d`で手動起動
