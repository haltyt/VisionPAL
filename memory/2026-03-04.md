## パルAR公開（JST 16:00〜19:30）

### 画像生成
- Nano Banana Pro（Gemini 3 Pro Image）でラーメンパル、Nano Banana 2（Gemini 3.1 Flash）で秋葉原パル＆オーロラパル生成
- Flash版はProより速い＆半額。モデル名: `gemini-3.1-flash-image-preview`
- Flash版スクリプト: `/tmp/gen_flash.py`（nano-banana-proのスクリプトのモデル名変えただけ）

### 動画生成
- Veo 3.1 Fast（`veo-3.1-fast-generate-preview`）でオーロラパル画像→動画化成功
- Image-to-Videoは `types.Image(image_bytes=bytes, mime_type="image/png")` で渡す
- DLは `alt=media` と `key=API_KEY` をパラメータで渡す（URI直接だと認証エラー）
- 生成スクリプト: `/tmp/gen_video4.py`

### 3Dモデル生成
- Meshy API v1（`/openapi/v1/image-to-3d`）でGLB生成。v2エンドポイントは404
- ラーメンパル(15.5MB)→オーロラパル(14.1MB)に差し替え
- Meshy-6はテクスチャベイクに時間かかる（99%で長時間止まる、全体20分程度）

### model-viewer版AR（成功）
- HTMLファイル1つ + GLB + ポスター画像で完結
- iPhoneではAR Quick Look（ARKitネイティブ）が全画面ポップアップで起動 → 高精度
- model-viewerはブラウザ内WebARではなく、ネイティブアプリにハンドオフする仕組み
  - Android → Scene Viewer（ARCore）
  - iOS → AR Quick Look（ARKit）
  - WebXR対応ブラウザ → WebXR Device API

### 8th Wall OSS版（苦戦）
- 8th Wallは2026-02-28サービス終了、エンジンバイナリがOSS配布（github.com/8thwall/engine）
- **重要**: 8th Wallエンジンは**8frame**（A-Frameフォーク版）でないと動かない！標準A-Frameだとxrwebコンポーネントが初期化失敗
- 8frame-1.3.0 + xrextras + xr.js + xr-slam.js の組み合わせでカメラAR動作確認
- エフェクト追加時にA-Frame標準版に差し替えたらカメラ起動しなくなった → 8frameに戻して解決
- インタラクティブエフェクト追加: タップでジャンプ、スピン、星フラッシュ、光の輪、絵文字リアクション

### 8th Wall vs model-viewer
- model-viewer: 3Dモデルを現実に置くだけ → ネイティブAR（高精度）。インタラクティブ性なし
- 8th Wall: ブラウザ内JS SLAM → 精度は劣るがインタラクティブ性◎（ゲームロジック、フェイスフィルター、イメージターゲット等）

### Tailscale Funnel設定
- パスベースルーティング: `/`=OpenClaw(18789), `/pal-ar`=model-viewer(8888), `/pal-ar-8thwall`=8th Wall版(8889)
- `tailscale serve --bg --set-path=/pal-ar http://172.19.0.2:8888` でパス追加
- Funnel再設定時にルートが443にリセットされる問題あり → serve設定後にfunnel再有効化
