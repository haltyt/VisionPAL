# VAD設計書 - voice_monitor v5

## 問題
v4では`SOUND_THRESHOLD=200`の固定RMS閾値のみで音声判定。環境ノイズが変動するとWhisperに無音/ノイズが送られ、幻聴（「字幕」「ご視聴ありがとう」等）が発生。

## v5の改善

### 1. 適応的エネルギーVAD
- 録音の**冒頭10フレーム(300ms)**をノイズレベルとして自動推定
- 閾値 = `max(ノイズRMS × 3.0, 100)` で動的に設定
- 環境が変わっても自動追従

### 2. ゼロ交差率(ZCR)チェック
- 高周波ノイズ（ファン音等）はZCRが高い（0.4超）
- 音声は通常ZCR 0.1-0.3程度
- `エネルギー閾値超え AND ZCR < 0.4` で音声フレーム判定

### 3. 発話区間長チェック
- 音声フレームが**5フレーム(150ms)以上**連続で検出されないと音声なしと判定
- 瞬間的なノイズ（物を置く音等）を除外

### 4. 幻聴フィルタ強化
- 既知パターンに追加（「ご覧いただき」「ラジオ」等）
- 1-2文字ひらがな/カタカナのみ → 除外
- 同一文字繰り返し（「ああああ」） → 除外
- 句読点・記号のみ → 除外

## フロー変更

```
v4: 録音 → RMS計算 → 閾値比較 → Whisper → 幻聴フィルタ
v5: 録音 → VAD解析(適応閾値+ZCR+発話長) → Whisper → 幻聴フィルタ(強化)
```

VADで弾けばWhisper APIコール自体を省略 → コスト削減

## パラメータ

| パラメータ | 値 | 説明 |
|---|---|---|
| VAD_FRAME_MS | 30ms | フレーム長 |
| VAD_ENERGY_MULTIPLIER | 3.0 | ノイズ比倍率 |
| VAD_MIN_ENERGY | 100 | 最低閾値 |
| VAD_ZCR_MAX | 0.4 | ZCR上限 |
| VAD_MIN_SPEECH_FRAMES | 5 | 最低発話フレーム |
| VAD_NOISE_FRAMES | 10 | ノイズ推定用フレーム数 |

## ファイル

- arm64/Jetson Nano対応

## チューニングガイド
- 幻聴がまだ出る → `VAD_ENERGY_MULTIPLIER`を4.0に上げる
- 声が拾えない → `VAD_MIN_SPEECH_FRAMES`を3に下げる
- ファンノイズで誤検出 → `VAD_ZCR_MAX`を0.3に下げる
